このトピックでは、ライブイベントを作成および管理する方法について説明します。 Live Module を開きます。

概要

Live モジュールを使用すると、ライブイベントをデスクトップデバイスとモバイルデバイスの両方に配信することができます。 Live モジュールにイベントの詳細を登録していただくと、オンサイトエンコーダを接続するためのエンコーダー設定情報が用意されます。 すべてのトランスコーディングはクラウド上で行われるため、ハードウェアベースのトランスコーダは必要ありません。

ライブ サブスクリプションに応じて、イベント、チャンネル、またはその両方を作成することができます。 通常、イベントは短い長さ、24回または定期的なイベントをストリーミングするために作成されます。 チャンネルは通常、7 / XNUMXライブイベントを配信するときに使用されます。

注: ライブストリーミングのサポートは Video Cloud Pro と Enterprise のパブリッシャーのみが利用できます。

必須条件

ライブイベントを配信するには、次の項目が必要です。

  • Live モジュールへのアクセス権を持つ Video Cloud Pro または Enterprise アカウント
  • イベントを放送するためのカメラ
  • TelestreamのWirecast or オープンブロードキャスターソフトウェアなど、お持ちのエンコーディングソフトウェアが必要です。

制限事項

  • Media モジュールで DVR を使用した Live ストリームを作成するためには、まずモジュール内で新規のリモートアセットを作成し、Live モジュールからの プレイリスト URL の最後に "playlist_dvr.m3u8" を追加します。
  • SSAI を使用した Live スト​​リームは現時点では Live モジュールからは作成ができません。この機能を使用した Live ストリームを配信するためには Media モジュールから行う必要があります。
  • Live 配信では20-30秒の遅延が経験されます。
  • ストリーミングは、イベント開始の30分以内に開始する必要があります（デフォルトレベルのアカウント設定）。
  • Internet Explorerを使用している場合、10以降のバージョンが必要です。
  • Live DVRを使用したスト​​リームは Brightcove Player SDK この場合、DVRの再生URLをリモートアセットに追加してから、SDKでそのリモートアセットURLを使用する必要があります。

ライブイベントの管理

アクセスするために Live モジュール、にログインする Video Cloud スタジオとクリック Live ナビゲーションヘッダーに表示されます。 ログイン後、あなたの Video Cloud アカウント名がページの右上隅に表示されます。 複数のアカウントをお持ちの場合は、アカウントスイッチャーのプルダウンを使用して、ご利用のアカウントを選択してください。 選択したアカウントが有効になっていない場合 Live、はじめに」Brightcove Liveビデオページが表示されます。[イベントの管理]ページには、3つのメインタブが表示されます。

  • Live - 進行中のライブイベントを一覧表示する
  • スタンバイ - スタンバイモードになっている 繰り返しイベント の一覧リスト

    • スタンバイ - アクティブにできるストリーム（静的エントリポイントのみ）

    • 完了 - Live ストリームが完了しました

    • 非アクティブ化済み - 繰り返しイベントは非アクティブ状態です
    • 非アクティブ - 繰り返しイベントは有効になっていますが、ストリーム開始は発生していません
    • 待機中 - RTMPストリームの待機中（エンコーダの開始）

新しいイベントを作成する

Live モジュールを使用してライブイベントを作成およびストリーミングするための完全なステップバイステップガイドについては ステップバイステップ： Live モジュールを Telestream Wirecast で利用する を参照してください。

新しいイベントを作成するには、次の手順を実行します。

  1. 「イベントの管理」ページで イベントの作成 をクリックします。 作成する Live イベントまたは Live チャンネルページが開きます。 イベントとチャンネルを作成するときのプロパティは同じです。 このドキュメントのスクリーンショットは、イベントの作成をカバーしています。
  2. イベントの詳細を入力してください：
    • イベント名 - イベントの名前。 ビデオがビデオアセットとして保存されている場合、この名前はMediaモジュールにも表示されます（最大255文字）。
    • 簡単な説明 - イベントの説明（最大250文字）
    • タグ - イベントに関連付けるためのタグ。 ビデオがビデオアセットとして保存されている場合、タグはビデオに割り当てられます（最大128文字および1200タグ）。
    • リファレンス ID - イベントのリファレンス ID。 ユニークである必要があります（最大150文字）
  3. アカウントにて必須なカスタムフィールドが表示されます。 カスタムフィールドに値を割り当てます。
  4. ソーシャル メディアへのストリーミングを許可 を 許可 に設定すると、イベントをソーシャルメディアプラットフォームへのストリーミングで利用できるようになります。 イベントのソーシャルメディアでの利用は、ストリーミングの開始後、または Social モジュール から可能です。
  5. ライブトランスコードプロファイル を 選択します。 ドロップダウンリストには、 Live 用のインジェストプロファイルが表示されます。 インジェストプロファイルは、作成されたライブレンディションの品質と数を制御します。 プロファイルの管理 リンクをクリックすると、別のブラウザタブでインジェストプロファイルエディタが開きます。 インジェストプロファイルエディタにアクセスするには、アカウント管理者である必要があります。 レンディション名 リンクをクリックして、選択したインジェストプロファイルを使用して作成されるレンディションを表示します。
    3つの層のビットレートと請求の詳細は次のとおりです。
    • 層XNUMXは最大XNUMX Kbpsを許可し、XNUMX x時間に課金されます
    • 層XNUMXは最大XNUMX Kbpsを許可し、XNUMX x時間に課金されます
    • 層XNUMXは最大XNUMX Kbpsを許可し、XNUMX x時間に課金されます

    注： Live インジェストプロファイル作成に関する情報については カスタム Live トランスコードプロファイルの作成 をご参照ください。

  6. 完了後、イベントを動画アセットに変換 を選択すると、終了後にイベントを VOD として Video Cloud のアカウントに保存できます。 このオプションを選択すると、選択した ライブトランスコードプロファイル での最高品質のレンディションが VOD 動画のソースとして使用されます。 VOD レンディションの生成にはアカウントのでフォールトのトランスコードプロファイルが使用されます。 ライブイベントの終了直後には VOD 動画がご利用できません。動画レンディションの生成が終了するまで、VOD 動画のご利用ができませんことにご留意ください。

    注：このオプションは、繰り返しイベントを使用する場合はサポートされていません。

  7. 詳細オプション をクリックして 詳細オプションを設定します。
  8. ストリーミング開始 をクリックしてライブイベントを開始します。調整室のページ が開き、エンコーダー設定で使用される ストリーミングエンドポイント（RTMP URL） と ストリーム名 を含むライブイベントの詳細が表示されます。

    注：ストリームを処理するのに約 XNUMX 分かかります。

    注：デフォルトでは、新しいイベントが作成されてからXNUMX分以内にエンコーダーを開始する必要があります。

ストリーミングを停止する ボタンをクリックしてイベントを終了します。

注：ライブストリームが完了したら、エンコーダーが閉じられているか停止していることを確認してください。 エンコーダーと Brightcove が接続されたままの場合、たとえデータが送信されていなくても、予期しない接続料金が発生する可能性があります。

定期的なイベントの概要

選択する 静的エントリポイント（SEP）を使用して、これを定期的な予定として作成します。 オプションを使用すると、同じエントリポイントのURLと再生URLを維持したまま、長時間実行可能なライブストリームをアクティブ化および非アクティブ化できます。 定期的なイベントを使用する場合、ストリーミングエンドポイントのURLとビデオIDは変更されません。

定期的なイベントを作成すると、Mediaモジュールでそのイベント用のビデオが自動的に作成されることに注意してください。 行う 次の例は正しくありません。 イベントが繰り返されるたびに必要になるため、このビデオを削除します。

制限事項

定期的なイベントを処理する場合、次の制限が適用されます。

  • . 完了後、イベントを動画アセットに変換 オプションは利用できません。 別の方法は、クリッピングを使用して、完全なライブイベントをVODとしてアーカイブすることです。 クリップは、ストリームの次のアクティブ化まで、または7日間のいずれか短い方で作成できます。 ストリームが再度アクティブになると、以前の録音セッションはサーバーから削除され、クリップできなくなります。
  • ジョブ設定は変更できません。現在、定期的に定期的なイベントを作成した設定に関連付けられています。
  • ストリームを有効にすると、イベントが利用可能になるまで60秒かかることがあります。 一部のエンコーダには再試行（Wirecast、Elementalなど）が組み込まれているため、エンコーダはエントリポイントが利用可能になるとすぐに再接続します。
  • 再接続時間 エンコーダの切断時の定期的なイベントにも適用されます。 もし 再接続時間 に達すると、定期的なイベントは非アクティブ状態に設定されます。 Brightcove ジョブが完了したときにストリームを非アクティブにし、 再接続時間 （この待機期間中に入力時間コストが発生した）場合には、受け入れ可能な時間枠まで待機する必要があります。
  • 定期的なイベントを無効にする前に、エンコーダを停止する必要があります。 アカウントには、有限数の定期的なイベントが割り当てられます。 制限に達すると、定期的なイベントの1つを停止してから新しいイベントを作成する必要があります。 反復イベントの最大数に達して別のイベントを作成しようとすると、エラーが戻されます。

定期的なイベントの操作

定期的なイベントを作成するには、 静的エントリポイント（SEP）を使用して、これを定期的な予定として作成します。 ライブイベントを作成してクリックすると イベントの作成.

繰り返しイベントが作成されると、そのイベントは スタンバイ 「イベントの管理」ページの「管理」タブをクリックします。

定期的な予定のステータスは、次のいずれかになります。

  • 非アクティブ - 定期的なイベントが作成され、ストリーミングが開始されたことがない
  • 待機中 - 定期的なイベントがアクティブになり、ストリームを待機している
  • ストリーミング - 定期的なイベントが進行中
  • 切断済み - 定期的なイベントは停止中です。
  • 非アクティブ化済み - 定期的なイベントが無効になっている
  • アーカイブ - 定期的なイベントがアーカイブされています

定期的なイベントの有効化

定期的なイベントを有効にすると、ジョブをストリーミングで利用できるようになります。 繰返しイベントは、エンコーダが開始される前にアクティブ化される必要があります。 定期的なイベントを有効にするには：

  1. スタンバイタブをクリックします。
  2. イベント名をクリックして、コントロールルームでイベントを開きます。
  3. アクティブ化をクリックします。
  4. エンコーダを起動します。

定期的なイベントがアクティブになるとスタンバイタブからLiveタブへ移動します。

定期的な予定の無効化

定期的なイベントを無効にすると、ストリーミングのためのジョブが無効になります。 イベントが終了したら定期的なイベントを無効にする必要があります。 定期的なイベントを無効にするには：

  1. Liveタブをクリックします。
  2. イベント名をクリックして、コントロールルームでイベントを開きます。
  3. 非アクティブ化をクリックします。
  4. エンコーダーを停止します。

定期的なイベントのアーカイブ

定期的なイベントをアーカイブすると、ジョブは永続的に非アクティブ化され、ストリーミングを有効にすることはできません。 最終的にイベントは削除されます。 ストリーミングが開始されたイベントのみをアーカイブできます。 定期的なイベントが作成され、ストリーミングが開始されなかった場合、 キャンセル ボタンが表示されます アーカイブ ボタンをクリックします。

定期的な予定をアーカイブするには：

  1. スタンバイタブをクリックします。
  2. イベントの行をクリックして選択します。
  3. アーカイブをクリックします。
